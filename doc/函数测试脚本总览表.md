# 函数测试脚本总览表

> 目的：用于说明 `functions/` 目录下各测试脚本的核心参数、测试对象与测试方法。

## 总览

| 模块 | 脚本 | 入口函数 | 主要参数（默认值） | 测试对象 | 测试方法 |
|---|---|---|---|---|---|
| 4g | `functions/4g/test_4g.py` | `test_4g` | `device=None, at_command='AT', network_test=False, timeout=30` | 4G 模组识别与 AT 通信能力 | 检测 4G 设备节点与串口状态；在启用 `network_test` 时执行 ping 连通性验证 |
| audio | `functions/audio/test_audio.py` | `test_audio` | `record_test=False, playback_test=False, duration=3, device=None, timeout=30` | 音频输入与输出能力 | 检查音频设备状态；根据参数执行 `aplay` 播放和 `arecord` 录音验证 |
| ble | `functions/ble/test_ble.py` | `test_ble` | `adapter='hci0', scan=False, constant_freq=False, timeout=30` | BLE 适配器状态与扫描能力 | 检查 `hciconfig`/`hcitool` 结果；在启用扫描时检索周边蓝牙设备 |
| csi | `functions/csi/test_csi.py` | `test_csi` | `device=None, capture_test=False, output=None, timeout=30` | CSI 摄像头检测与图像采集能力 | 通过 `v4l2-ctl`/`media-ctl` 检测设备；在启用采集时调用 `ffmpeg`/`fswebcam` 抓图 |
| dsi | `functions/dsi/test_dsi.py` | `test_dsi` | `check_output=False, resolution=None, timeout=30` | DSI 显示接口状态 | 读取显示接口状态和分辨率信息并进行规则判定 |
| eeprom | `functions/eeprom/test_eeprom.py` | `test_eeprom` | `bus=None, address=80, read_test=False, write_test=False, write_protect_test=False, timeout=30` | EEPROM 设备识别与读写能力 | 通过 `i2cdetect` 扫描总线；按参数执行 `i2cget`/`i2cset` 读写验证 |
| fan | `functions/fan/test_fan.py` | `test_fan` | `device=None, pwm_value=128, rpm_test=False, timeout=30` | 风扇控制与转速反馈能力 | 读取并写入风扇 PWM/转速相关节点后进行结果判定 |
| gpio | `functions/gpio/test_gpio.py` | `test_gpio` | `pin, mode='output', value=None, timeout=10` | GPIO 导出、方向和值控制 | 通过 sysfs GPIO 节点执行导出、方向配置与电平读写测试 |
| hailo | `functions/hailo/test_hailo.py` | `test_hailo` | `device=None, inference_test=False, model=None, timeout=60` | Hailo 设备识别与推理链路能力 | 检测 Hailo 设备与运行时环境；在启用推理测试时加载模型执行验证 |
| hdmi | `functions/hdmi/test_hdmi.py` | `test_hdmi` | `check_output=False, resolution=None, interface=None, timeout=30` | HDMI/DP 显示接口状态 | 检测显示接口是否存在；按参数执行输出状态与分辨率检查 |
| i2c | `functions/i2c/test_i2c.py` | `test_i2c` | `bus=None, address=None, timeout=10, scan_all=False` | I2C 总线与设备探测能力 | 扫描 I2C 总线并检测目标地址设备是否存在 |
| lora | `functions/lora/test_lora.py` | `test_lora` | `device=None, interface=None, frequency=915, tx_test=False, timeout=30` | LoRa 模组识别与收发能力 | 检查 LoRa 设备节点/接口状态；在启用发送测试时验证发射链路 |
| maskrom | `functions/maskrom/test_maskrom.py` | `test_maskrom` | `device=None, list_devices=False, timeout=30` | Maskrom 模式检测能力 | 使用 `rkdeveloptool` 枚举并识别 Maskrom 设备 |
| network | `functions/network/test_eth.py` | `test_eth` | `ip, interface=None, timeout=10, count=4, iperf3=False` | 以太网连通性与吞吐能力 | 自动检测网卡后执行 `ping` 连通性测试；在启用 `iperf3` 时执行吞吐测试 |
| rtc | `functions/rtc/test_rtc.py` | `test_rtc` | `rtc_device='/dev/rtc0', set_time=None, sync_network=False, timeout=10` | RTC 设备可用性与时间读写能力 | 检查 RTC 节点状态；按参数执行时间设置与网络时间同步校验 |
| ssd | `functions/ssd/test_ssd.py` | `test_ssd` | `device=None, full_test=False, test_size='100M', timeout=60` | SSD 识别与性能能力 | 识别存储设备；按参数执行 `dd`/`fio` 性能测试 |
| uart | `functions/uart/test_uart.py` | `test_uart` | `port, baudrate=115200, timeout=10, loopback=False` | UART 串口通信能力 | 打开串口收发数据并验证通信结果；在启用环回模式时执行环回测试 |
| usb | `functions/usb/test_usb.py` | `test_usb` | `port=None, speed_test=False, timeout=10` | USB 设备识别与速度能力 | 通过 `lsusb` 枚举 USB 设备；在启用测速时执行读写速度测试 |
| wifi | `functions/wifi/test_wifi.py` | `test_wifi` | `ssid, password=None, interface=None, timeout=30` | Wi-Fi 扫描、连接与连通性能力 | 使用 `nmcli`/无线命令连接指定 SSID，并执行连通性验证 |
| wifi_halo | `functions/wifi_halo/test_wifi_halo.py` | `test_wifi_halo` | `interface=None, ssid=None, connect_test=False, timeout=30` | Wi-Fi Halow 设备识别与连接能力 | 检测 Halow 接口状态；在启用连接测试时连接指定 SSID 并验证连通性 |

## 说明

- 表格参数来源于各脚本 `test_*` 函数签名，默认值对应默认执行行为。
- “测试方法”列为实现逻辑归纳，便于快速理解，不等同于逐行代码说明。
- 如需运行示例参数，请优先参考 `cases/*_case.json` 及脚本 CLI `--help` 输出。
